# Настройка Яндекс.Диска для бота

В боте используется Яндекс.Диск для хранения видео: админ создаёт категории (папки), загрузчик загружает видео в папки категорий, после того как получатель отправит ссылку-результат — видео удаляется с Диска.

## 1. Регистрация приложения и получение OAuth-токена

1. Перейдите в [OAuth-сервис Яндекса](https://oauth.yandex.ru/).
2. Нажмите **«Зарегистрировать новое приложение»**.
3. Заполните:
   - **Название** — любое (например, «Бот для видео»).
   - **Права доступа** — отметьте:
     - **Яндекс.Диск** → «Доступ к папке приложения на Диске» или **«Доступ к Диску на чтение и запись»** (рекомендуется).
   - **Callback URI** для «Публичное приложение» можно указать `https://oauth.yandex.ru/verification_code` (для ручного получения токена).
4. Сохраните приложение. Скопируйте **ID приложения (ClientID)**.

## 2. Получение токена для своего аккаунта

1. Откройте в браузере ссылку (подставьте свой `ClientID`):
   ```
   https://oauth.yandex.ru/authorize?response_type=token&client_id=ВАШ_CLIENT_ID
   ```
2. Войдите в аккаунт Яндекса и разрешите доступ.
3. После перенаправления в адресной строке будет фрагмент вида `access_token=...`. Скопируйте значение `access_token` (без слова `access_token=` и до следующего `&`).

Либо используйте [страницу выдачи токена](https://yandex.ru/dev/disk/api/concepts/access.html) (если у приложения включена «Поддержка паролей»).

## 3. Настройка .env

В файле `.env` в корне проекта укажите:

```env
YANDEX_DISK_TOKEN=ваш_токен_из_шага_2
YANDEX_DISK_ROOT_PATH=/bot_videos
```

- **YANDEX_DISK_TOKEN** — OAuth-токен (обязательно).
- **YANDEX_DISK_ROOT_PATH** — путь к корневой папке на Яндекс.Диске, внутри которой создаются папки категорий. По умолчанию `/bot_videos`. Можно изменить, например, на `/Проект`.

## 4. Перезапуск бота

После сохранения `.env` перезапустите бота, чтобы подхватить новые переменные.

## Как это работает в боте

1. **Админ** создаёт категорию → на Яндекс.Диске создаётся папка `YANDEX_DISK_ROOT_PATH/Название_категории` (или используется существующая с таким именем).
2. **Загрузчик** выбирает категорию и загружает видео → файл попадает в папку этой категории на Диске, в БД сохраняется ссылка и путь к файлу.
3. **Получатель (дистрибьютор)** получает видео и ссылку на Яндекс.Диск, после отправки ссылки-результата бот удаляет этот файл с Диска.

Файлы хранятся в том аккаунте Яндекс, для которого получен токен; квота — квота этого аккаунта.

**Видео, загруженные вручную:** положите файл в папку категории на Диске — путь должен совпадать с тем, что использует бот: `YANDEX_DISK_ROOT_PATH/Название_категории` (например `/bot_videos/Название_категории`). При нажатии «Получить видео» бот подхватит новые файлы из папок категорий и выдаст их.
